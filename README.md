# ♟️ LLM Chess Hallucination Analysis

This repository contains an interactive **Streamlit application** and dataset used to evaluate the reliability of Large
Language Models (LLMs) in explaining chess positions.

The project focuses on identifying and analyzing **hallucinations** in natural-language explanations generated by
different models.

---

## 🚀 Overview

Large Language Models show strong potential for **educational applications**, especially in chess e-learning. However,
their ability to generate **accurate and reliable explanations** remains limited.

This project:

* Evaluates multiple state-of-the-art LLMs
* Identifies different types of hallucinations
* Introduces a **weighted scoring framework**
* Provides an **interactive interface** for exploration

## 📊 Dataset Description

### 1. `aggregated_scores.xlsx`

Contains normalized and aggregated model scores across all reviewed studies (Art. 1–4).
Scores are on a 0–100 scale. The *Mean Score* column is a weighted average by number of appearances.

---

### 2. `dataset_scored.xlsx`

Contains annotated outputs from multiple LLMs.

Each model has its own sheet:

```
Data <MODEL_NAME>
```

### Key Columns:

* `puzzle_id` → links to a puzzle in the PGN file
* `move` → move referenced in the explanation
* `original_claim` → model-generated explanation
* `hallucination_type` → annotated error type
* `fixed_claim` → corrected version of the explanation
* `potential_impact` → educational impact of the error

---

### 3. `puzzles_with_moves.pgn`

Contains the **ground truth chess positions and move sequences**.

Each puzzle includes:

* Initial board position
* Correct sequence of moves

Used in the app to:

* Replay positions
* Align explanations with real moves

## 🧠 Hallucination Evaluation Framework

The scoring system is based on three components:

### 1. Category Weights (W_c)

| Category        | Weight |
|-----------------|--------|
| move_legality   | 8      |
| piece_placement | 6      |
| material        | 5      |
| piece_relations | 4      |
| geometry        | 3      |
| reasoning       | 2      |
| other           | 1      |

---

### 2. Severity Multipliers (M_s)

| Severity | Multiplier |
|----------|------------|
| major    | 3          |
| medium   | 2          |
| minor    | 1          |

---

### 3. Decay Factor (δ)

* δ = 0.7
* Applied for multiple hallucinations within the same (puzzle, category) group
* Reduces the impact of repeated similar errors

## ⚠️ Key Findings

* LLMs exhibit **two main failure modes**:

    * Structural hallucinations (board misunderstanding)
    * Reasoning hallucinations (incorrect logic)

* Even top models:

    * Produce **confident but incorrect explanations**
    * Lack deterministic reasoning

* This creates **pedagogical risk** in learning environments
